# Contributing to Anthropic Bug Bounty Guide

Thank you for helping make this guide better! 🙏

## How to Contribute

### 1. Fix Documentation
- Fix typos, unclear explanations, broken links
- Improve existing methodology guides
- Add missing information

### 2. Add New Content
- New vulnerability testing methodology
- New tools or scripts
- Redacted example reports (with permission)
- Tips from your bug bounty experience

### 3. Report Issues
- Found incorrect information? Open an issue
- Have a question? Open a discussion

## Contribution Process

1. **Fork** this repository
2. **Create a branch:** `git checkout -b fix/typo-in-readme` or `feature/add-xss-methodology`
3. **Make your changes**
4. **Commit:** `git commit -m "docs: fix typo in hackerone-standards.md"`
5. **Push:** `git push origin your-branch`
6. **Open a Pull Request**

## What NOT to Submit

❌ Do NOT submit:
- Actual vulnerability details (even "fixed" ones — coordinate with Anthropic first)
- Personal data or credentials
- Content that could help with unauthorized testing
- Competitor analysis targeting other companies

✅ DO submit:
- Educational methodology
- General security testing techniques
- Tools that work within authorized scope
- Anonymized/redacted case studies

## Code of Conduct

- Be respectful and constructive
- Help beginners — everyone starts somewhere
- No gatekeeping — security should be accessible
- English or Hinglish both welcome 🇮🇳

## Questions?

Open a [GitHub Discussion](../../discussions) — we're happy to help!
